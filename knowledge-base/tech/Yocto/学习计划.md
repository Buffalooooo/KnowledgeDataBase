## 第一章：认识 Yocto（概念入门）

> **目标**：理解 Yocto 是什么、能做什么、核心术语体系。
> 📝 **本章笔记** → [[Yocto-概述]]

### □ 1.1 Yocto 是什么
- [x] Yocto 是一个开源协作项目，提供模板、工具和方法来创建定制 Linux 系统
- [x] 它不是 Linux 发行版，而是**构建系统**（Build System）
- [x] 适用于嵌入式设备、IoT、工业控制等场景

### □ 1.2 核心术语体系
- [x] 理解 **Poky**：Yocto 的参考构建系统，包含 BitBake + OE-Core
- [x] 理解 **BitBake**：任务执行引擎（类似 make 但更强大）
- [x] 理解 **Recipe（配方）**：`.bb` 文件，描述如何构建一个软件包
- [x] 理解 **Layer（层）**：`.bb` 和 `.bbappend` 文件的集合
- [x] 理解 **Machine（机器）**：目标硬件平台的定义
- [x] 理解 **Distribution（发行版）**：系统级配置策略
- [x] 理解 **Image（镜像）**：最终生成的系统镜像

### □ 1.3 Yocto 构建工作流概览
- [x] 理解整体流程：获取源码 → 配置 → 编译 → 打包 → 生成镜像
- [x] 理解 Yocto 与 Buildroot 的区别
- [x] 了解 Yocto 的版本命名规则（如 scarthgap, mickledore）

### □ 验收清单
- [x] 能用自己的话解释「Yocto 是什么」
- [x] 能说出至少 5 个核心术语的含义
- [x] ✅ 创建笔记 [[Yocto-概述]]

## 📖 附录：常用命令速查

### 环境与配置
| 命令 | 用途 |
|------|------|
| `bitbake -e <recipe>` | 展开所有变量 |
| `bitbake -s \| grep <pkg>` | 搜索包 |
| `bitbake-layers show-recipes` | 显示所有 Recipe |
| `bitbake-layers show-layers` | 显示所有 Layer |

### 构建控制
| 命令 | 用途 |
|------|------|
| `bitbake <target> -c clean` | 清理任务输出 |
| `bitbake <target> -c cleanall` | 完全清理（含下载） |
| `bitbake <target> -C compile` | 强制重新编译 |
| `bitbake -k <target>` | 出错继续执行 |

### 依赖分析
| 命令 | 用途 |
|------|------|
| `bitbake -g <target>` | 生成依赖图 |
| `bitbake -g -u depexp <target>` | 图形化依赖查看 |

### 文件查询
| 命令 | 用途 |
|------|------|
| `oe-pkgdata-util list-pkgs` | 列出所有已构建包 |
| `oe-pkgdata-util find-path /bin/ls` | 查找文件所属包 |
| `oe-pkgdata-util read-pkg <pkg>` | 查看包信息 |

### 常见问题排查
| 问题 | 排查方法 |
|------|---------|
| 构建失败 | 查看 `tmp/log/cooker/<machine>/console-latest.log` |
| Recipe 解析错误 | `bitbake -S none <recipe>` 查看解析过程 |
| 包找不到 | `bitbake -s \| grep <name>` 或检查 Layer 路径

### 常用调试命令速查

```bash
# 环境与配置
bitbake -e <recipe>                  # 展开所有变量
bitbake -s | grep <package>          # 搜索包
bitbake-layers show-recipes          # 显示所有 Recipe
bitbake-layers show-layers           # 显示所有 Layer

# 构建控制
bitbake <target> -c clean            # 清理任务输出
bitbake <target> -c cleanall         # 完全清理（含下载）
bitbake <target> -C compile          # 强制重新编译
bitbake -k <target>                  # 出错继续执行

# 依赖分析
bitbake -g <target>                  # 生成依赖图
bitbake -g -u depexp <target>        # 图形化依赖查看

# 文件查询
oe-pkgdata-util list-pkgs           # 列出所有已构建包
oe-pkgdata-util find-path /bin/ls   # 查找文件所属包
oe-pkgdata-util read-pkg <pkg>      # 查看包信息
```

### 常见问题排查

| 问题 | 排查方法 |
|------|---------|
| 构建失败 | 查看 `tmp/log/cooker/<machine>/console-latest.log` |
| Recipe 解析错误 | `bitbake -S none <recipe>` 查看解析过程 |
| 包未包含在镜像中 | 检查 `IMAGE_INSTALL` 和 `RRECOMMENDS` |
| 文件冲突 | 检查 `FILES:` 变量和包拆分 |
| 网络超时 | 配置 `PREMIRRORS` 或使用本地源 |
