### 2026-07-13：Git status 显示中文文件名乱码 / 换行符导致文件显示已修改

**现象**：
1. `git status` 中中文文件名显示为转义码，如 `\346\246\202\350\277\260.md`
2. 无实质改动的文件因换行符差异（LF vs CRLF）反复显示为 `modified`

**原因**：
1. 中文文件名乱码：Git 默认对非 ASCII 文件名转义（`core.quotepath=true`）
2. 换行符问题：Windows（CRLF）与 Linux（LF）换行符差异导致 Git 误判文件已修改

**解决方案**：

**方案一**：仅解决中文显示
```bash
git config --global core.quotepath false
```

**方案二**：一劳永逸，同时解决换行符问题（推荐）
```bash
# 在仓库根目录创建 .gitattributes
echo "* text=auto" >> .gitattributes
git add .gitattributes
git commit -m "chore: 添加 .gitattributes 统一换行符处理"
```

### 2026-07-14：PowerShell 启动时 profile.ps1 加载报错

**现象**：每次打开 PowerShell 终端时显示：
```
无法加载文件 ...\profile.ps1，因为在此系统上禁止运行脚本。
```

**原因**：PowerShell 默认执行策略为 `Restricted`，禁止运行任何脚本文件（包括用户自定义的 profile.ps1）

**解决方案**：
```powershell
# 将执行策略改为 RemoteSigned（允许本地脚本，远程脚本需签名）
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 2026-07-15：Yocto do_fetch 速度慢（80KB/s），配置宿主机代理后解决

**现象**：VM（Ubuntu）中执行 `bitbake core-image-minimal`，`do_fetch` 下载速度仅 80KB/s 左右，`linux-yocto` 等大包下载极其缓慢。

**原因**：VMware NAT 模式下，VM 网络流量不经过宿主机 VPN/代理，导致直连 GitHub 等源速度极慢。

**解决方案**：

**宿主机（Windows）**：在代理软件设置中开启「允许局域网连接」（Allow LAN），确认代理端口（本例为 7890）

**VM（Ubuntu）**：
```bash
# 1. 配置代理环境变量（写入 ~/.bashrc 永久生效）
echo 'export http_proxy=http://192.168.112.1:7890' >> ~/.bashrc
echo 'export https_proxy=http://192.168.112.1:7890' >> ~/.bashrc
source ~/.bashrc

# 2. 让 bitbake 继承代理环境变量（写入 local.conf）
echo 'BB_ENV_PASSTHROUGH_ADDITIONS = "http_proxy https_proxy"' >> ~/Yocto/poky/build/conf/local.conf

# 3. 重新构建（已下载过的包会命中 DL_DIR 缓存）
bitbake core-image-minimal
```

> **注意**：`192.168.112.1` 是宿主机在 VMnet8（NAT）上的 IP，`7890` 是代理端口，根据实际情况修改。

### 2026-07-15：Windows Git push 到 GitHub 失败（连接超时）

**现象**：`git push` 时报错：
```
fatal: unable to access 'https://github.com/...': Failed to connect to github.com port 443
```

**原因**：GitHub 直连被限制。开了 VPN 后 Git 未配置代理端口，仍然走直连导致超时。

**解决方案**：
```powershell
# 配置 Git 代理（端口根据 VPN 软件实际设置修改）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 重新推送
git push origin main

# 如需取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```
