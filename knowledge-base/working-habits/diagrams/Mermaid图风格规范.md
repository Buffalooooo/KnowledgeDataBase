# Mermaid 图风格规范

> 知识库中的所有 Mermaid 图遵循以下规范，保持风格统一。

---

## 1. 颜色主题

| 层次 | 用途 | 填充色 | 边框色 |
|------|------|--------|--------|
| 元数据层 | 配方、配置、核心库 | `#E8F5E9` 浅绿 | `#66BB6A` 绿 |
| 构建层 | BitBake、Recipe、Layer | `#E3F2FD` 浅蓝 | `#42A5F5` 蓝 |
| 系统层 | 内核、文件系统 | `#FFF3E0` 浅橙 | `#FFA726` 橙 |
| 应用层 | 用户软件 | `#FCE4EC` 浅粉 | `#EF5350` 红 |
| 硬件抽象 | Machine、Distribution | 白色虚线边框 | `#78909C` 灰蓝 |

## 2. 图表类型选择

| 表达内容 | 推荐图类型 | 方向 |
|----------|-----------|------|
| 架构分层（组件归属） | `graph TB` 或 `graph BT` | 从上到下 或 从下到上 |
| 流程/流水线 | `graph LR` | 从左到右 |
| Layer 叠加/优先级 | `graph BT` | 从下到上 |
| 数量/对比 | 表格（非 Mermaid） | — |

## 3. 节点样式

- **形状**：`[ ]` 方框
- **文字**：中文为主，关键英文术语加粗或保留原名
- **小字说明**：用 `<br/><small>...</small>` 换行
- **样式定义**：使用 `classDef` + `class` 统一管理，避免 inline

```mermaid
graph LR
    classDef green fill:#E8F5E9,stroke:#66BB6A,stroke-width:2px

    A[节点名称<br/><small>说明文字</small>]:::green
    B[另一个节点]:::green
    A --> B
```

## 4. 分组规范

- 使用 `subgraph` 对同类节点分组，分组名带图标前缀
- 分组示例：`subgraph META["📦 元数据层"]`

| 图标 | 用途 |
|------|------|
| 📦 | 数据/配置/元数据 |
| 🔧 | 工具/构建过程 |
| ⚙️ | 硬件/平台 |
| 🐧 | Linux 系统 |
| 🚀 | 应用/部署 |
| 🔀 | 依赖/叠加关系 |
| 🖼️ | 镜像产物 |
| 🛠️ | SDK/工具链 |

## 5. 流程图阶段标记

当流程图有多个阶段时，使用 `subgraph` 拆分为阶段，命名格式：

```
PHASE1["阶段❶ 名称"]
PHASE2["阶段❷ 名称"]
```

用中文数字 `❶❷❸❹` 而不是数字 `1. 2.`。

## 6. 高亮规则

关键节点加高亮色（黄色 `#FFEB3B`）：

```mermaid
graph LR
    classDef highlight fill:#FFEB3B,stroke:#F57F17,stroke-width:2px

    A[关键步骤]:::highlight
```

用于：
- 流程图中用户最关注的操作（如 `do_compile`、`do_image`）
- 架构图中当前讨论的核心组件

## 7. 统一格式

- 箭头用 `-->`（带箭头实线）
- 分组名中英双语（中文为主，英文关键术语加粗）
- 节点文字超过 8 个字考虑换行 `<br/>`
